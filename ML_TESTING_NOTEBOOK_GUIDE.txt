===============================================================================
KEY METRICS EXPLAINED
===============================================================================

CLASSIFICATION METRICS:

Accuracy:
  - Percentage of correct predictions
  - Range: 0-1 (0% to 100%)
  - Higher is better

Precision:
  - Of positive predictions, how many were correct?
  - Range: 0-1
  - Important for minimizing false positives

Recall:
  - Of actual positives, how many were correctly identified?
  - Range: 0-1
  - Important for minimizing false negatives

F1 Score:
  - Harmonic mean of Precision and Recall
  - Range: 0-1
  - Balanced metric for imbalanced datasets

AUC-ROC:
  - Area Under the Receiver Operating Characteristic curve
  - Range: 0-1 (0.5 is random, 1.0 is perfect)
  - Measures ability to distinguish between classes


REGRESSION METRICS:

MSE (Mean Squared Error):
  - Average of squared differences
  - Lower is better
  - Penalizes large errors heavily

RMSE (Root Mean Squared Error):
  - Square root of MSE
  - Same units as target variable
  - More interpretable than MSE

MAE (Mean Absolute Error):
  - Average of absolute differences
  - Lower is better
  - More robust to outliers than MSE

R² Score:
  - Coefficient of determination
  - Range: -∞ to 1 (1.0 is perfect)
  - Proportion of variance explained by model


PERFORMANCE METRICS:

Inference Time:
  - Time to make predictions
  - Measured in seconds
  - Lower is better for real-time systems

Throughput:
  - Samples processed per second
  - Higher is better
  - Important for batch processing

GOOD MODEL PERFORMANCE:
  ✓ Classification: Accuracy > 0.80, F1 > 0.75
  ✓ Regression: R² > 0.70, RMSE < 10% of target range
  ✓ Inference: < 100ms for single prediction
  ✓ Throughput: > 100 samples/second

AREAS FOR IMPROVEMENT:
  ⚠ Accuracy < 0.70 - Model may need retraining
  ⚠ Large gap between precision/recall - Imbalanced dataset
  ⚠ High residuals in regression - Consider feature engineering
  ⚠ Slow inference - Model size or complexity issue
