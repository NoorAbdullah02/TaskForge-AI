PROJECT OVERVIEW
TaskForge AI is an enterprise-ready task and team intelligence platform built 
to support workspace-based onboarding, team collaboration, project management, 
notifications, AI-driven insights, and operational reporting.

Repository URL: https://github.com/NoorAbdullah02/TaskForge-AI
License: ISC
Current Version: 1.0.0
Minimum Node.js: 20.0.0+


KEY FEATURES
1. WORKSPACE & ONBOARDING
   - Workspace-based architecture with invite links
   - Registration codes for team members
   - Admin invite emails for workspace setup
   - First-user super admin bootstrap
   - Role-based workspace access control

2. TEAM COLLABORATION
   - Pending workspace join requests
   - Approval workflow for member management
   - Real-time socket notifications with socket.io
   - Team management and member oversight

3. PROJECT MANAGEMENT
   - Project, Sprint, Epic, Story, Task management
   - Task assignment and tracking
   - Milestone management
   - Time tracking and logging
   - Kanban board and Gantt chart views

4. EMPLOYEE MANAGEMENT
   - Attendance tracking
   - Leave management
   - Calendar scheduling
   - Time tracker
   - Notification workflows

5. AI & INTELLIGENCE FEATURES
   - Productivity prediction & analytics
   - Project delay prediction
   - Burnout risk assessment
   - Resource optimization recommendations
   - Project success probability prediction
   - Deadline estimation
   - AI-powered task generation from descriptions
   - Meeting summarization & action item extraction
   - AI Copilot for workspace assistance
   - Enterprise AI insights

6. REPORTING & DASHBOARDS
   - Executive Dashboard
   - PM Dashboard
   - Team Leader Dashboard
   - Employee Dashboard
   - Owner Dashboard
   - Super Admin Console
   - Project Intelligence Dashboard
   - Reports with export capabilities

7. INFRASTRUCTURE
   - Email queuing and trigger service
   - Background job processing with BullMQ
   - Redis caching and queue management
   - Real-time notifications
   - File upload and management
   - Knowledge Base system

INSTALLATION & SETUP

PREREQUISITES:
  - Node.js v20 or higher
  - npm v10 or higher
  - PostgreSQL database
  - Redis server
  - Python 3.8+ (for ML service)
  - Optional: Email provider (Brevo, Resend, etc.)

STEP-BY-STEP INSTALLATION:

1. Clone Repository:
   git clone https://github.com/NoorAbdullah02/TaskForge-AI-.git
   cd "TaskForge AI"

2. Install Dependencies:
   npm install --prefix Backend --include=dev
   npm install --prefix frontend --include=dev
   cd ml-service && pip install -r requirements.txt

3. Database Setup:
   - Create PostgreSQL database: taskforge
   - Update DB_URL in Backend/.env with your credentials
   - Run migrations: npm run db:push --prefix Backend

4. Configure Environment Variables:
   - Copy template env vars to Backend/.env (see ENVIRONMENT VARIABLES above)
   - Create frontend/.env with VITE_API_BASE_URL

5. Start Services (in separate terminals):
   Terminal 1 - Backend:
     npm run dev --prefix Backend

   Terminal 2 - Frontend:
     npm run dev --prefix frontend

   Terminal 3 - ML Service:
     cd ml-service
     python run.py

6. Access Application:
   - Frontend: http://localhost:5173
   - Backend API: http://localhost:4000/api
   - ML Service: http://localhost:8000
   - Swagger Docs: http://localhost:4000/api-docs (if available)

PRODUCTION BUILD:
   npm run build                        [From root directory]

COMMON ISSUES & TROUBLESHOOTING
================================================================================

Issue: "Cannot connect to database"
  Solution: Verify PostgreSQL is running, check DB_URL in .env

Issue: "ML Service connection failed"
  Solution: Start ML service on port 8000, check ML_SERVICE_URL in Backend/.env

Issue: "Redis connection error"
  Solution: Start Redis server, verify REDIS_URL in .env

Issue: "Email not sending"
  Solution: Check BREVO_API_KEY and email credentials in .env

Issue: "Socket.io not connecting"
  Solution: Verify backend socket.io setup, check CORS origins

Issue: "Frontend API calls failing"
  Solution: Verify VITE_API_BASE_URL, check backend CORS settings
